## Overview

`LRU` (Linear Recurrent Unit) is a simplified state space model that learns complex diagonal dynamics directly without explicit discretization. It uses learnable complex eigenvalues parameterized in the polar form (radius and phase), making it both simple and effective.

LRU uses FFT-based convolution for parallel training and supports efficient autoregressive inference.

### Parameters

<ParamField path="d_model" type="int" required>
  Model dimension (hidden size / number of features).
</ParamField>

<ParamField path="d_state" type="int" required>
  State dimension (internal SSM state size).
</ParamField>

<ParamField path="r_min" type="float" default={0}>
  Minimum radius for Lambda (eigenvalue) initialization. Eigenvalues are initialized uniformly on an annulus between `r_min` and `r_max`.
</ParamField>

<ParamField path="r_max" type="float" default={1}>
  Maximum radius for Lambda (eigenvalue) initialization. Setting `r_max=1` constrains eigenvalues to have magnitude ≤ 1, ensuring stability.
</ParamField>

<ParamField path="max_phase" type="float" default={"2 * math.pi"}>
  Maximum phase for Lambda initialization. Phases are initialized uniformly between 0 and `max_phase`.
</ParamField>

## Example Usage

### Basic Usage

```python
from lrnnx.models.lti import LRU
import torch

# Create LRU model
model = LRU(d_model=64, d_state=64)

# Training: parallel mode
x = torch.randn(2, 128, 64)  # (batch, length, features)
y = model(x)
print(y.shape)  # torch.Size([2, 128, 64])
```

### Autoregressive Inference

```python
# Allocate cache for inference
cache = model.allocate_inference_cache(batch_size=2)

# Generate sequence token by token
outputs = []
for t in range(100):
    x_t = torch.randn(2, 64)  # (batch, features)
    y_t, cache = model.step(x_t, cache)
    outputs.append(y_t)

# Stack outputs
y_seq = torch.stack(outputs, dim=1)  # (batch, length, features)
print(y_seq.shape)  # torch.Size([2, 100, 64])
```

### Custom Eigenvalue Initialization

```python
# Initialize with tighter radius range for more stable dynamics
model = LRU(
    d_model=128,
    d_state=128,
    r_min=0.9,     # Minimum radius close to 1
    r_max=0.999,   # Maximum radius close to 1
    max_phase=math.pi  # Restrict phase to [0, π]
)
```

## Key Features

### Learned Complex Dynamics

LRU directly learns complex eigenvalues in polar form:

```python
Λ = exp(-exp(nu_log) + 1j * exp(theta_log))
```

where:
- `nu_log`: Controls the radius (magnitude) of eigenvalues
- `theta_log`: Controls the phase (angle) of eigenvalues

This parameterization:
- **Ensures stability**: Constraining radius to [0, 1] keeps eigenvalues in unit circle
- **Learns time scales**: Different radii correspond to different decay rates
- **Captures oscillations**: Phases enable modeling of oscillatory dynamics

### No Explicit Discretization

Unlike S4/S4D/S5, LRU does not use explicit discretization methods (ZOH, bilinear, etc.). Instead:
- Eigenvalues Λ are learned directly in discrete-time form
- The model learns appropriate time scales through `nu_log` and `theta_log`
- A learnable normalization factor `gamma_log` scales the input projection

### Normalization Factor

LRU includes a learnable normalization:

```python
gamma = exp(gamma_log)
B_normalized = B * gamma
```

The paper suggests `gamma` can also be computed as:
```python
gamma = sqrt(1 - |Λ|²)
```
which maintains signal energy, but making it learnable yields similar results.

### System Matrices

| Matrix | Shape | Dtype | Description |
|--------|-------|-------|-------------|
| nu_log | `(d_state,)` | float32 | Log of radial component |
| theta_log | `(d_state,)` | float32 | Log of angular component |
| B_re, B_im | `(d_state, d_model)` | float32 | Real and imaginary parts of B |
| C_re, C_im | `(d_model, d_state)` | float32 | Real and imaginary parts of C |
| D | `(d_model,)` | float32 | Skip connection |
| gamma_log | `(d_state,)` | float32 | Log of normalization factor |

## Architecture Details

### Initialization

```python
# Eigenvalue parameters: uniform on annulus [r_min, r_max] × [0, max_phase]
u1, u2 = rand(N), rand(N)
nu_log = log(-0.5 * log(u1 * (r_max² - r_min²) + r_min²))
theta_log = log(max_phase * u2)

# Input/output projections: Glorot initialization
B_re, B_im ~ Normal(0, 1/sqrt(2*d_model))
C_re, C_im ~ Normal(0, 1/sqrt(d_state))

# Skip connection
D ~ Normal(0, 1)

# Normalization factor
Λ = exp(-exp(nu_log) + 1j * exp(theta_log))
gamma_log = log(sqrt(1 - |Λ|²))
```

### Recurrent Update

For autoregressive inference:

```python
# State update
state_{t+1} = Λ * state_t + B_norm @ x_t

# Output (take real part)
y_t = Real(C_complex @ state_{t+1}) + D * x_t
```

### Training Forward Pass

For parallel training:

```python
# Compute kernel: K[i] = Λ^i for i = 0, ..., L-1
K = Λ^{0:L}

# Convolve: Y = K ⊛ (B @ X) 
y = FFT_conv(K, B_norm @ x) + D * x

# Where ⊛ denotes convolution and @ is matrix multiplication
```

## Performance Notes

- **Simple**: Fewer parameters and hyperparameters than S4/S4D
- **No discretization overhead**: Learns dynamics directly in discrete time
- **Fast training**: FFT-based parallel convolution
- **Efficient inference**: Pre-computed matrices for autoregressive generation
- **Stable**: Polar parameterization naturally constrains eigenvalues

## Comparison with Other Models

| Model | Discretization | State Matrix | Normalization | Complexity |
|-------|----------------|--------------|---------------|------------|
| S4 | Internal (DPLR) | Diagonal + Low-Rank | Implicit | High |
| S4D | Via `disc` param | Diagonal | Implicit | Medium |
| S5 | ZOH/Bilinear/Dirac | Complex Diagonal | Via dt | Medium |
| **LRU** | **None (learned)** | **Complex Diagonal** | **Learnable γ** | **Low** |

## When to Use LRU

Choose LRU when:
- You want the simplest possible SSM architecture
- You prefer learned discretization over explicit methods
- You need stable long-term dependencies (eigenvalues in unit circle)
- You want minimal hyperparameter tuning
- Your task benefits from learning time scales end-to-end

## Notes

- LRU uses `discretization="no_discretization"` as it doesn't use explicit discretization
- The polar parameterization (radius, phase) naturally ensures eigenvalue stability
- Setting `r_max=1` constrains eigenvalues to unit circle, guaranteeing bounded dynamics
- The skip connection D is per-feature, not a full matrix (unlike S5)
